# Лабораторная работа №1: GitHub Webhooks и автоматическое развёртывание

## Описание
В данной лабораторной работе реализован обработчик webhook для автоматического развёртывания приложения **catty-reminders-app** при получении push-событий от GitHub. Обработчик написан на Python с использованием Flask, запускается как systemd-сервис и взаимодействует с FRP для обеспечения внешней доступности.

## Структура репозитория
- `webhook_handler.py` — основной скрипт обработчика webhook.
- `requirements.txt` — зависимости Python для обработчика (Flask).
- `README.md` — данный файл с описанием.

## Принцип работы
1. GitHub отправляет POST-запрос на `http://webhook.simonov.course.prafdin.ru` при каждом push в любой ветке репозитория.
2. Обработчик проверяет заголовок `X-GitHub-Event` и игнорирует события, отличные от `push`.
3. Запускается фоновый процесс `deploy`, который:
   - Клонирует или обновляет код приложения в директории `/opt/catty-app`.
   - Устанавливает зависимости из `requirements.txt` (если файл существует).
   - Перезапускает сервис `catty-app` через systemd.
4. Обработчик сразу возвращает GitHub статус `202 Accepted`, не дожидаясь завершения развёртывания.
5. Приложение становится доступно по адресу `http://app.simonov.course.prafdin.ru`.

## Требования к окружению
- Ubuntu / любой Linux с systemd
- Python 3.8+
- Установленный FRP-клиент (настроен на проксирование портов 8080 и 8181)
- Git
